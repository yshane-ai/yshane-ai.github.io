Pointer Network详解与Pytorch实践

**Pointer network用于解决从输入序列中选择合适的item作为输出的问题，例如经典旅行商问题，RL中目标选择，文献总结等。**在旅行商问题（traveling salesman problem，TSP）中，给定若干城市的位置信息，旅行家需要规划途径城市的顺序来达到能遍历每个城市但总路径和最小的目的。因此，学会理解和运用pointer network是十分必要的。

1 概述

一个pointer network通常由以下三部分组成：

  • encoder模块
  • decoder模块
  • attention模块

在使用pointer network时，一个简易版本的旅行商问题可以描述为：

  • 输入：一系列城市的坐标$(x, y)$
  • 输出：一个城市的序列，代表着对城市的最优遍历路径

整体流程图如下图所示，每个城市的坐标作为一个token，我们向encoder（通常使用RNN）输入一个token的序列，获得每个token的embedding，同时获得encoder的最终隐状态。

将encoder的最终隐状态输入到decoder模块中，每次产生一个向量，我们称之为query。query使用attention机制计算与encoder输出的每个embedding的注意力值，选择注意力值最大的token作为当前输出。重复以上操作，直到输出“停止”的token。

以上基于深度学习的方法，我们可以使用较低的复杂度便可以获得对旅行商问题的的一个近似最优解。

2 如何使用pointer network？

现在我们使用一个例子来对pointer network进行讲解，问题是从一堆发票中提取发票开具方的公司地址。

我们将以上问题建模为：从半结构化的文本中抽取有效的信息。什么是半结构化的文本？我们对文本进行如下的划分：

  • 结构化文本：给定了一个模板，我们可以根据模板定位想要的信息的位置，例如某种简历模板，所有使用该模板的应聘者姓名，性别等信息都在同一位置
  • 无结构文本：信息杂乱无序，例如写一篇文章来对自己进行介绍
  • 本结构化文本：文本采用结构化的方式，但是模板不清楚，无法对信息直接进行定位，例如不同的简历模板，名字的位置也不相同，有的在左上角，有的在中上部分

基于以上原因，我们希望使用模型提取半结构化文本中的信息时，需要使模型同时捕捉词条的语义和位置信息。

通过使用OCR提取出一系列的目标文本后，我们向pointer network输入这些词条以及他们的2D位置信息，然后获得我们想要得到的输出。

比如说，如下所示，我们希望可以找到以下发票中发票开具方公司地址信息：

当把整个序列输入到encoder模块后，我们得到最终的隐状态向量，并把它作为decoder模块的第一个输入。借用最普通的attention思想，每一步decoder产生一个输出并计算与输入序列中token的距离，选择距离最近的token作为输出。

然后，我们将这个token的embedding当作下一步decoder的输入，计算下一步的输出。

重复以上过程，我们可以得到一条完整的输出序列，通过比较其和正确输出序列的误差，利用反向传播的方式去优化pointer network，我们可以获得最终的模型。

3 数据及制作

3.1 数据集

我们面临的任务是需要从一堆使用不同模板的发票中抽取信息。为了产生稍复杂同时也很真实的数据，我们使用了python的Faker包来随机产生一些地址和文本。

数据集的信息包括：

  • 发票接收方的公司的名字和地址
  • 发票的一些元信息，例如日期和id号码
  • 发票开具方公司的地址，电话，电子邮箱地址和名称
  • 发票的内容，例如描述，发票金额和数量等

数据集中只包含了1000份文本，对机器学习和深度学习方法而言相对小了一些。

3.2 预处理和标签制作

通常使用机器学习方式，都需要对数据进行预处理。在这个任务中，我们需要从一堆发票图片中提取出相关的文字内容。**我们可以使用一项名为Tesseract的OCR技术来做文字提取。 ** 我们可以知道文本中的地址的位置，但是我们需要先将其转换为序号标签，方便pointer network的训练。

我们将真个数据集按照6:2:2的比例划分为训练/验证/测试集。也就是说，我们使用600条样本用来做训练，200条用来做验证，剩下200条用来做测试。

3.3 数据清洗

在做NLP任务时，我们需要首先考虑使用哪种embedding技术？例如，使用预训练的技术（如word2vec或fasttext）需要使用固定的预料库，并且需要在训练NLP任务前对其预先进行训练，在训练NLP模型期间token的embedding不会再发生变化。这里我们选择使用一种端到端的架构直接同时训练embedding和目标模型，embedding会跟着模型一起更新。

考虑到OCR识别的准确率等原因，我们选用一个小的语料库，以每个字母为一个单位为每个单词产生一个token，将一个token的序列作为LSTM网络的输入，获得每个token的embedding以及LSTM的最终隐状态。

4 样本

为了加速训练，我们使用大一点的batch size，这需要我们将数据转化为tensor格式再组成batch。方便进行训练，我们需要完成以下三个步骤：

  1. 对每个单词做character level的划分

    1. 0用于做padding
    2. 单词的每个character都被映射为一个整数

  2. 每一页上每个单词的位置使用2D表示

  3. 训练标签是每页发票上表示地址的单词的序号，使用0做padding

综合以上想法，我们得到三个多维tensor：

  1. 单词

    1. 第一维表示数据集的大小
    2. 第二位表示每条样本中单词的最大数量
    3. 第三维表示每个单词的最大长度

  2. 位置

    1. 第一维表示数据集的大小
    2. 第二位表示每条样本中单词的最大数量
    3. 第三维中前两个数字$(x,y)$表示目前单词相对文本左上角在图片中的像素坐标，后两个数字$(x_, y_)$表示右下角相对于左上角在图片中的像素坐标

  3. 目标

    1. 第一维表示数据集的大小
    2. 第二位表示每条样本中单词的最大数量

5 建模

我们可以将整个模型拆分成以下三部分：

  1. Word embedder
  2. Encoder
  3. Decoder

5.1 Word Embedder

  1. 首先构建一个embedding层对每个字母做embedding
  2. 将embedding输入到LSTM网络中
  3. 获得最终的隐状态，作为当前单词的embedding
class TokenEmbedder(nn.Module):
    INIT_NORMALIZE_DENOMINATOR = (
        1000
    )  # Same order of magnitude that the width in pixels

    def __init__(self, n_characters: int, embedding_dim: int, position_dim: int):
        super().__init__()
        self.chars_embedding_layer = nn.Embedding(
            num_embeddings=n_characters, embedding_dim=embedding_dim, padding_idx=0
        )

        self.word_embedding_layer = nn.LSTM(
            input_size=embedding_dim, hidden_size=embedding_dim, num_layers=1
        )

        self.dropout_word_embedding = nn.Dropout(0.5)

        self.normalize_position = nn.Parameter(
            self.INIT_NORMALIZE_DENOMINATOR * torch.ones(4), requires_grad=True
        )
        self.position_embedding_layer = nn.Linear(4, position_dim)
        self.dropout_position = nn.Dropout(0.5)

    def forward(self, words: torch.tensor, positions: torch.tensor) -> torch.tensor:
        character_embeddings = self.chars_embedding_layer(words)

        batch_size, n_words, n_characters, dim = character_embeddings.shape
        character_embeddings = character_embeddings.reshape(
            batch_size * n_words, n_characters, dim
        ).transpose(0, 1)

        _, (hn, _) = self.word_embedding_layer(character_embeddings)

        word_embeddings = self.dropout_word_embedding(hn.transpose(0, 1))
        word_embeddings = word_embeddings.reshape(batch_size, n_words, dim)

        positions = positions / self.normalize_position
        positions_embedding = torch.sigmoid(self.position_embedding_layer(positions))
        positions_embedding = self.dropout_position(positions_embedding)

        raw_token_embeddings = torch.cat([positions_embedding, word_embeddings], dim=2)
        return raw_token_embeddings

5.2 Encoder

Encoder的模型较为简单，就是按照阅读的顺序（从左上到右下）将每个单词的embedding和position embedding输入到LSTM网络中，每一步获得对当前单词的一个contextualized embedding，以及最终的一个隐状态作为整个文本的embedding。

class Encoder(nn.Module):
    def __init__(self, embedding_dim: int, position_dim: int):
        super().__init__()

        self.embedding_dim = embedding_dim

        self.lstm = nn.LSTM(
            input_size=embedding_dim + position_dim,
            hidden_size=embedding_dim,
            num_layers=1,
            bidirectional=True,
        )
        self.dropout = nn.Dropout(0.5)

    def forward(self, raw_token_embeddings: torch.tensor):
        batch_size, _, _ = raw_token_embeddings.shape
        token_embeddings, (encoded_document, _) = self.lstm(
            raw_token_embeddings.transpose(0, 1)
        )
        token_embeddings = self.dropout(token_embeddings.transpose(0, 1))
        encoded_document = self.dropout(
            encoded_document.reshape(batch_size, 2 * self.embedding_dim)
        )
        return token_embeddings, encoded_document

5.3 Decoder

我们对最终的隐状态做展开，并将输出的隐状态和之前每个单词获得的contextualized embedding进行比较，选择最相像的token作为当前步的输出。

class PointerNetwork(nn.Module):
    def __init__(self, embedding_dim: int, max_seq_len: int):
        super().__init__()
        self.embedding_dim = embedding_dim
        self.max_seq_len = max_seq_len

        self.lstm_cell = nn.LSTMCell(
            input_size=2 * embedding_dim, hidden_size=embedding_dim
        )
        self.dropout = nn.Dropout(0.5)

        self.reference = nn.Linear(embedding_dim, 1)
        self.decoder_weights = nn.Linear(embedding_dim, embedding_dim)
        self.encoder_weights = nn.Linear(2 * embedding_dim, embedding_dim)

    def attention(
        self, token_embeddings: torch.tensor, hx: torch.tensor
    ) -> torch.tensor:
        batch_size, n_tokens, hidden_dim = token_embeddings.shape

        decoder_query = self.decoder_weights(hx)
        token_embeddings = self.encoder_weights(token_embeddings)

        decoder_query = decoder_query.repeat(
            n_tokens, 1, 1
        )  # n_token x batch_size x embedding_dim
        decoder_query = decoder_query.transpose(
            0, 1
        )  # batch_size x n_token x embedding_dim
        comparison = torch.tanh(decoder_query + token_embeddings)
        probabilities = torch.log_softmax(
            self.reference(comparison).reshape(batch_size, n_tokens), 1
        )
        return probabilities

    def forward(
        self, token_embeddings: torch.tensor, encoded_document: torch.tensor
    ) -> Tuple[torch.tensor, List[int]]:
        batch_size, _, _ = token_embeddings.shape
        overall_probabilities = []
        batch_identifier = torch.arange(batch_size).type(torch.LongTensor)

        peak_indices = []
        decoder_input = encoded_document
        for step in range(self.max_seq_len):
            hx, cx = self.lstm_cell(decoder_input)
            hx = self.dropout(hx)
            probabilities = self.attention(token_embeddings, hx)

            _, peak_idx = probabilities.max(dim=1)
            decoder_input = token_embeddings[batch_identifier, peak_idx, :]

            overall_probabilities.append(probabilities)
            peak_indices.append(peak_idx)

        overall_probabilities = torch.stack(overall_probabilities).transpose(0, 1)
        peak_indices = torch.stack(peak_indices).t()
        return overall_probabilities, peak_indices

5.4 模型

将以上三个部分结合起来可以获得我们最终的模型，首先利用OCR获得每页发票上的文本信息，利用word embedder模块计算每个单词的embedding，获得当前页的embedding序列并作为encoder的输入，然后使用decoder接收encoder的输出作为输入，生成一系列token作为输出。
