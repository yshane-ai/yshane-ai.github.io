XGBoost和LightGBM详细对比与Python实践

2022-04-08
ML

Table of Contents

在做kaggle比赛时，XGBoost和LightGBM成为了大众竞赛者的首选方案，但他们有什么不同呢？其实从各方面来看。LightGBM都胜过XGBoost，本文将针对他们之间的优劣进行深入比较。

1 背景

XGBoost在2016年被首次提出并夺得多个数据竞赛的冠军，但次年Microsoft便开源了LightGBM（Light Gradient Boosting Machine），使用20%的训练时间便可以达到与XGBoost相同的水准。

通过本篇文章，最后你会收获到：

  • 如何为分类或者回归任务搭建一个LightGBM模型
  • XGBoost和LGBM的结构差异
  • 如何使用提前结束和测试集来防止过拟合
  • LGBM的内置缺失值处理
  • 如何做交叉验证

2 XGBoost vs. LightGBM

XGBoost和LightGBM均属于集成算法。他们使用一系列弱学习模型，例如决策树，来构建更复杂的强学习模型。

在XGBoost中，决策树每次以层为单位进行构造，如下图所示：

因为XGBoost一直以层为单位进行构造，直到达到最大深度，导致某些分裂增益很小的节点也一直被分裂，很容易产生不必要的节点，进而造成训练资源的一定浪费。

相反，LightBoost以叶子结点为单位进行构造，如下图所示：

每次LightGBM只选择分裂增益最大的叶子节点进行分裂，直到最后达到最大叶子数。这也是最初LightGBM被提出时就可以在时间效率上远远碾压XGBoost的原因。

如下图所示，LightGBM的训练效率比XGBoost高了一个量级。

LGBM同时对连续特征使用了直方图（histogram binning）算法，基本思想是：先把连续的浮点特征值离散化成$k$个整数，同时构造一个宽度为$k$的直方图。在遍历数据的时候，根据离散化后的值作为索引在直方图中累积统计量，当遍历一次数据后，直方图累积了需要的统计量，然后根据直方图的离散值，遍历寻找最优的分割点。这样做有两个优点：

  1. 内存占用更小，计算代价更小
  2. 利用差加速可以用非常小的代价获得兄弟叶子的直方图

因此，XGBoost借鉴LGBM的思想，提供了一个hist版本的XGBoost，但性能还没能追上LGBM，如下图所示。

3 模型初始化和目标

XGBoost和LGBM都有两种调用API，一种是原生API，另一种是与sklearn相兼容的API。本文针对后一种API进行讲解。

Aklearn API提供了LGBMRegressor和LGBMClassifier两个类，并且配有'fit/predict/predict_proba'方法：

import lightgbm as lgbm  # standard alias

clf = lgbm.LGBMClassifier(objective="binary")  # or 'mutliclass'
reg = lgbm.LGBMRegressor()  # default - 'regression'

objective参数指明了任务的类型，一般包括binary，multiclass和regression三种任务。

>>> reg.fit(X, y)
LGBMRegressor()

4 控制模型的复杂度

决策树的数量直接影响了模型的精度，你可以通过调整n_eatimators这个参数来设置决策树的数量，如下我们将在Kaggle TPS March数据集上构造1000棵决策树：

tps_march = pd.read_csv("data/train.csv")
tps_march.head()

# Encode categoricals
>>> X_enc = pd.get_dummies(X)

>>> clf = lgbm.LGBMClassifier(
...    objective="binary", n_estimators=1000, random_state=1121218
...  )
>>> clf.fit(X_enc, y)

增加更多的树容易使模型过拟合，为了防止过拟合，我们可以增加树数量的同时降低学习率。

>>> clf = lgbm.LGBMClassifier(
...    objective="binary", n_estimators=3000, learning_rate=0.1, random_state=1121218
...)

>>> clf.fit(X_enc, y)
LGBMClassifier(n_estimators=3000, objective='binary', random_state=1121218)

4 Early stopping

在集成模型中，每棵树都是根据已有模型的误差进行构建，因此当模型的准确率在某个迭代次数后不再提升时，我们可以适当地停止训练。

为了实现这个目标，LGBM提供了early_stopping_rounds参数用于提前停止训练。例如，我们将值设置为100，那么模型的准确率在过去100轮都没有上升时模型就停止训练。

Early stopping仅有在fit方法输入测试集eval_set时有效，此时测试集用于对模型的效果进行评估。

from sklearn.model_selection import tXGBClassifiertest_split

X_train, X_eval, y_train, y_eval = train_test_split(X_enc, y, test_size=0.1)

clf = lgbm.LGBMClassifier(objective="binary", n_estimators=10000)
eval_set = [(X_eval, y_eval)]

clf.fit(
    X_train,
    y_train,
    eval_set=eval_set,
    early_stopping_rounds=100,
    eval_metric="binary_logloss",
)

5 Baseline

让我们按照以上流程搭建一个模型作为baseline，同时再使用XGBoost也搭建一个相似的模型对比效果：

import xgboost as xgb

X, y = tps_march.drop("target", axis=1), tps_march[["target"]].values.flatten()

6 缺失值处理

LGBM的直方图算法用于处理数据中的缺失值和特征。在TPS March数据集中，一共有19个类，我们使用one-hot方法来实现对缺失值的处理操作，这次我们使用LGBM的内置处理并与XGBoost进行比较：

X, y = tps_march.drop("target", axis=1), tps_march[["target"]].values.flatten()

# Extract categoricals and their indices
cat_features = X.select_dtypes(exclude=np.number).columns.to_list()
cat_idx = [X.columns.get_loc(col) for col in cat_features]

# Convert cat_features to pd.Categorical dtype
for col in cat_features:
    X[col] = pd.Categorical(X[col])

# Unencoded train/test sets
X_train, X_eval, y_train, y_eval = train_test_split(
    X, y, test_size=0.2, random_state=4, stratify=y
)

为了指定类别特征，我们向fit方法传入类别特征的特征序号。

# Model initialization is the same
eval_set = [(X_eval, y_eval)]

_ = lgbm_clf.fit(
    X_train,
    y_train,
    categorical_feature=cat_idx,  # Specify the categoricals
    eval_set=eval_set,
    early_stopping_rounds=150,
    eval_metric="logloss",
    verbose=False,
)

preds = lgbm_clf.predict_proba(X_eval)
loss = log_loss(y_eval, preds)
print(f"LGBM logloss with default cateogircal feature handling: {loss:.5f}")

可以发现，使用LGBM内置的方法在精度和速度上均处于领先。

7 交叉验证

交叉验证可以用于防止模型过拟合，通常是把数据集分为$k$份，其中在每次训练时，都会选择其中一份当做测试集，其他数据当做训练集，通过最后求取每次训练的测试集误差来判断模型是否过拟合并选择是否结束训练。

import time

from sklearn.model_selection import StratifiedKFold

N_SPLITS = 7
strat_kf = StratifiedKFold(n_splits=N_SPLITS, shuffle=True, random_state=1121218)

scores = np.empty(N_SPLITS)
for idx, (train_idx, test_idx) in enumerate(strat_kf.split(X, y)):
    print("=" * 12 + f"Training fold {idx}" + 12 * "=")
    start = time.time()

    X_train, X_val = X.iloc[train_idx], X.iloc[test_idx]
    y_train, y_val = y[train_idx], y[test_idx]
    eval_set = [(X_val, y_val)]

    lgbm_clf = lgbm.LGBMClassifier(n_estimators=10000)
    lgbm_clf.fit(
        X_train,
        y_train,
        eval_set=eval_set,
        categorical_feature=cat_idx,
        early_stopping_rounds=200,
        eval_metric="binary_logloss",
        verbose=False,
    )

    preds = lgbm_clf.predict_proba(X_val)
    loss = log_loss(y_val, preds)
    scores[idx] = loss
    runtime = time.time() - start
    print(f"Fold {idx} finished with score: {loss:.5f} in {runtime:.2f} seconds.\n")
