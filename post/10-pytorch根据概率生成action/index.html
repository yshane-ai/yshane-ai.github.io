Pytorch根据概率生成action

2020-04-04
RL Pytorch

Table of Contents

  • 随机numpy方法-np.random.choice（不推荐）
  • 随机torch方法
    • torch.distributions.Categorical
    • torch.multinomial
  • 贪心方法 - torch.max() / torch.topk()
  • 连续动作
  • 总结

在强化学习（Reinforcement Learning）的Actor-Critic算法中，Actor生成每个action的概率，然后我们根据这个概率来选择我们的action。

probs = Actor(state,action) # probs.shape = (action_size)

随机numpy方法-np.random.choice（不推荐）

当probs只是一维向量的时候，我们可以用numpy 的 np.random.choice()函数来获得我们的随机action a, replace指定能否对同一个action a 重复抽样。

import numpy as np
action_size = 1
a = np.random.choice(actions, action_size, p=probs, replace=False)
logp = p[a].log()

但是如果我们用cuda训练的话，还需要在numpy array和torch tensor 之间的转换。此外如果是batch训练时候，probs从一维变成二维tensor。这时候就没法使用np.random.choice了，因为它的input 必须是1-D array。

# probs.shape = (batch_size, action_size)

随机torch方法

torch.distributions.Categorical

probs = policy_network(state)
dist = torch.distributions.categorical.Categorical(probs)
action = dist.sample()
next_state, reward, done, info = env.step(action)
loss = -m.log_prob(action) * reward
loss.backward()

torch.multinomial

更加通用的做法是torch自身的torch.multinomial函数。

torch.multinomial(input, num_samples,replacement=False, out=None) → LongTensor

返回一个张量，每行包含从input相应行中定义的多项分布中抽取的num_samples个样本。 [注意]:输入input每行的值不需要总和为1 (这里我们用来做权重)，但是必须非负且总和不能为0。 当抽取样本时，依次从左到右排列(第一个样本对应第一列)。 如果输入input是一个向量，输出out也是一个相同长度num_samples的向量。如果输入input是有 mm行的矩阵，输出out是形如m×nm×n的矩阵。 如果参数replacement 为 True, 则样本抽取可以重复。否则，一个样本在每行不能被重复抽取。 参数num_samples必须小于input长度(即，input的列数，如果是input是一个矩阵)。

所以代码修改如下, 由于高维没法直接切片（slice）， 用torch.gather()函数取对应action的概率：

import torch
num_a = 1
a = torch.multinomial(probs, num_a, replacement=False)
logp = torch.gather(probs, index = a, dim = 1)
logp = logp.log()

可以看到两种方法都支持取多个action，也就是我们可以很方便的运用beam search。

num_a = beam_search_size # number of the action we choose

贪心方法 - torch.max() / torch.topk()

以上是随机的方法，对于确定性（Deterministic）或者说 greedy method 取值就更简单了, PyTorch 的最大值函数都自带了输出index。

只取一个最大值的时候：

prob, a = torch.max(probs, dim = 1)
logp = prob.log()

beam search取多个最大值时用torch.topk():

# torch.topk(input, k, dim=None, largest=True, sorted=True, out=None) -> (Tensor, LongTensor)
prob, a = torch.topk(probs, num_a, dim=1)
logp = prob.log()

连续动作

假如我们输出的动作是一个连续的动作，我们让神经网络输出期望(mu)和方差（sigma），根据mu和sigma使用torch.distributions.normal.Normal(mu, sigma)来构建一个正态分布，然后从中采用得出一个再-2到2之间的值。

class Net(nn.Module):
    ***
    def forward(self, x):
        x = self.net(x)
        mu = F.tanh(self.mu(x)) * 2
        sigma = F.softplus(self.sigma(x)) + 0.001
        return mu, sigma

class Agent():
    def select_action(self, x):
        with torch.no_grad():
            mu, sigma = self.net(x)
            dis = torch.distributions.normal.Normal(mu, sigma)
            a = dis.sample()
        return a.item()

总结

在RL中根据概率probs 选择行动 action a的时候，一般有随机stochastic和贪心greedy两种方法。

在随机中推荐使用 torch.multinomial()，贪心方法推荐使用 torch.topk() 。二者都可向下兼容只取一个action。
